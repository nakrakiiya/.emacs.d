Pure Prolog package

Author(s): The CLIP Group.

This library package allows the use of pure Prolog in a Ciao module/program. It is based on the fact that if an engine module is imported explicitly then all of them have to be imported explicitly. The engine modules are:

  • engine(arithmetic)

    Arithmetic.

  • engine(atomic_basic)

    Basic predicates handling names of constants.

  • engine(attributes)

    Attributed variables.

  • engine(basic_props)

    Basic data types and properties.

  • engine(basiccontrol)

    Control constructs/predicates.

  • engine(data_facts)

    Fast/concurrent update of facts.

  • engine(exceptions)

    Exception and Signal handling.

  • engine(io_aux)

    Message printing primitives.

  • engine(io_basic)

    Basic input/output.

  • engine(prolog_flags)

    Changing system behaviour and various flags.

  • engine(streams_basic)

    Basic file/stream handling.

  • engine(system_info)

    Gathering some basic internal info.

  • engine(term_basic)

    Basic term manipulation.

  • engine(term_compare)

    Comparing terms.

  • engine(term_typing)

    Extra-logical properties for typing.

Note that if any of these modules is explicitely imported in a program then the language defaults to Pure Prolog, plus the functionality added by the modules explicitely imported.

It is recommended that if you explicitely import an engine module you also use this package, which will guarantee that the predicate true/0 is defined (note that this is the only Ciao builtin which cannot be redefined).

Usage and interface

  • Library usage:
    :- use_package(pure). or :- module(...,...,[pure]).

Known bugs and planned improvements

  • Currently, the following builtin predicates/program constructs cannot be redefined, in addition to true/0: (->)/2 (,)/2 (+)/1 if/3
